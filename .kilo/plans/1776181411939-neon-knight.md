## Issue 2: Simplified Receiver Flow

### Current Flow ( Complex)
1. Receiver taps "I am a Receiver"
2. Receiver enters "Delivery Code" (6-digit code from sender)
3. System verifies code
4. Receiver scans traveler's QR code
5. Delivery confirmed

### Requested Flow (Simplified)
The receiver is the person receiving the package. They already know they're the receiver:
1. Receiver taps "I am a Receiver" 
2. Show their device's delivery QR code (auto-displayed)
3. Scan traveler's QR code to confirm receipt
4. Done - no manual code entry needed

### Changes Needed

#### Frontend (ReceiverScanScreen.tsx)
- [ ] Remove the "Delivery Code" input field for receivers
- [ ] Auto-display receiver's delivery QR code (from sender/generator)
- [ ] Keep only the QR scanner to scan traveler's QR code
- [ ] Streamline flow: Open → Show QR → Scan → Confirm

#### Backend (verify/add if needed)
- [ ] Ensure receiver's QR code is accessible without entering code
- [ ] Use authenticated user's ID or stored receiver verification

### User Stories Affected
- Receiver receiving package at destination
- No login/signup required (unauthenticated flow still works)
- Simpler UX - just scan traveler's code to confirm
