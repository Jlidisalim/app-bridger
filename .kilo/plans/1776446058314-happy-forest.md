# Implementation Plan: Design Updates (Steps 3–5)

## Context
Steps 1 (sender info/gallery in DealDetails) and 2 (multi-image in PackageDetails) are already completed.  
Three updates remain:

1. **Step 3 – RouteSelectionScreen**: Add airport/city autocomplete with worldwide suggestions.
2. **Step 4 – ReceiverDetailsScreen**: Replace free‑form phone country code entry with a CountryPicker.
3. **Step 5 – PricingScreen**: Remove the "Verify Receiver Code" section entirely.

---

## Step 3 — Airport Autocomplete in RouteSelectionScreen

### 3.1 Add airport dataset
- Create `src/data/airports.ts` exporting a constant `AIRPORTS`.
- Populate with ~10k IATA airports (fields: `iata`, `city`, `country`). Source: OpenFlights or a similar public-domain list. Include all local and international airports.

### 3.2 Build autocomplete component
- Create `src/components/AirportAutocomplete.tsx` (or implement inline).
- Props: `label`,value`, `onChange`, `placeholder`.
- Maintain local state: `query`(text), `showSuggestions`(bool), `selected`(bool).
- Filter `AIRPORTS` by `city`, `country`, or `iata` containing the query (case‑insensitive). Limit results (e.g., top 5–7).
- Render a dropdown `FlatList` positioned below the input while focused and `query` non‑empty.
- On suggestion press, call `onChange` with the selected **city name** (e.g., "New York") and clear suggestions.
- Allow free‑form entry if user dismisses dropdown or types full text without selecting.

### 3.3 Integrate into RouteSelectionScreen
- Replace both `Input` components (for "From" and "To") with two `AirportAutocomplete` instances.
- Keep the existing icon, label, and layout; ensure `from` and `to` state values are updated from the autocomplete.
- The `onNext` payload remains unchanged (`{ from, to, departureDate }`).

## Step 4 — CountryPicker for Receiver Phone

### 4.1 Expand country list (optional but recommended)
- Replace the limited hard‑coded `COUNTRIES` array in `CountryPicker.tsx` with a comprehensive list (~250 countries).
- Create `src/data/countries.ts` exporting `COUNTRIES` of type `Country[]` (with `name`, `flag`, `code`, `dialCode`). Source: restcountries.com or similar.
- Update `CountryPicker.tsx` to import this list (or keep local if preferred).

### 4.2 Modify ReceiverDetailsScreen
- Import `CountryPicker` and `Country` type.
- Add state:
  ```ts
  const [selectedCountry, setSelectedCountry] = useState<Country>(DEFAULT_COUNTRY);
  const [localPhone, setLocalPhone] = useState('');
  ```
- Parse incoming `senderReceiver?.phone` (if any) into `selectedCountry` & `localPhone` by matching dial codes; if parsing fails, leave defaults.
- Replace the single phone `Input` with a row:
  ```tsx
  <View style={styles.phoneRow}>
    <CountryPicker selectedCountry={selectedCountry} onSelect={setSelectedCountry} />
    <TextInput
      style={styles.phoneInput}
      placeholder="Phone number"
      value={localPhone}
      onChangeText={setLocalPhone}
      keyboardType="phone-pad"
    />
  </View>
  ```
- In `onNext`, combine: `const fullPhone = `${selectedCountry.dialCode}${localPhone}`;` and pass that as `phone`.
- Add necessary styles for `phoneRow` (flex row, border, radius) and `phoneInput` (flex: 1, padding).

### 5.1 Delete verification UI & state
